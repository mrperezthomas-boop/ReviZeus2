# 01 — CONSTITUTION DU PROJET RÉVIZEUS

## Usage IA
Lis ce document en **premier** pour comprendre la raison d'être de RéviZeus. Il définit ce qui ne peut pas être remis en cause : vision, positionnement, promesse utilisateur, architecture conceptuelle. Toute proposition qui violerait ces principes doit être signalée comme hors-doctrine.

## 1. Vision fondamentale

RéviZeus est une **application RPG éducative immersive** destinée à l'entraînement cérébral long terme, combinant :
- **Mythologie** (panthéons évolutifs)
- **IA adaptative** (IAristote)
- **Pédagogie moderne**
- **Gamification profonde**
- **Progression narrative structurée**

> L'objectif n'est pas simplement d'apprendre.
> L'objectif est l'**élévation cognitive durable**.

## 2. Positionnement

**RéviZeus n'est PAS :**
- Une app de QCM classique
- Une app scolaire froide
- Un simple jeu gamifié

**RéviZeus EST :**
Un RPG éducatif épique en style olympien chibi coloré, drôle et immersif, où la progression académique est intégrée à une quête mythologique évolutive.

## 3. Promesse utilisateur

- Progression sans plafond
- Récompenses abondantes
- Difficulté adaptative
- Immersion narrative
- Montée en maîtrise intellectuelle

## 4. Architecture conceptuelle

```
Application
│
├── IAristote Core (Cœur IA pédagogique)
├── Engine RPG (Progression & récompenses)
├── Module Panthéon (Dynamique divine)
├── Module Pédagogique (Cours, QCM, Résumés)
├── Dashboard principal
├── Module Réglages
└── Infrastructure Cloud & Sauvegarde
```

### IAristote Core
**Rôle :**
- Génération de résumés
- Création de QCM et d'exercices
- Évaluation intelligente
- Adaptation de la difficulté
- Commentaires pédagogiques

**Modes :**
- Adaptatif activé / désactivé
- Humour réglable
- Intensité pédagogique variable

### Module Panthéon
Phase 1 actuelle = Panthéon Grec (10 dieux). Phases 2 et 3 prévues : Nordique, Égyptien.

Chaque panthéon doit avoir :
- Son lore propre
- Son arc narratif
- Son système de progression
- Son boss final
- Sa conclusion mythologique

## 5. Game loop

### Boucle principale
1. Étudier (cours / résumé)
2. S'entraîner (exercices)
3. Valider (QCM)
4. Recevoir XP + récompenses
5. Monter de niveau divin
6. Débloquer lore / quêtes
7. Boss hebdomadaire
8. Répéter avec difficulté ajustée

### Boucle long terme
Maîtrise d'un dieu → complétion du panthéon → boss final narratif → transition vers panthéon suivant.

### Récompenses
XP, badges divins, titres, animations, lore caché, éléments cosmétiques, avatars, effets UI. **Pas de plafond.** Progression infinie, prestige possible.

## 6. Contraintes pédagogiques

### Objectifs cognitifs
- Mémorisation active
- Compréhension profonde
- Répétition espacée
- Application pratique
- Auto-évaluation

### Principes pédagogiques
- Difficulté progressive
- Feedback immédiat
- **Erreur = apprentissage**
- Variation des formats
- Adaptation IA dynamique

### Types de contenu
- Résumés synthétiques
- QCM progressifs
- Exercices ouverts

## 7. Règles identitaires immuables

Ces règles ne peuvent pas être remises en question par une IA sans validation humaine explicite :

1. **Portrait fullscreen obligatoire** (exception unique : mode Aventure en paysage)
2. **Kotlin + XML + ViewBinding** uniquement
3. **Jetpack Compose interdit**
4. **10 dieux = 10 matières** (panthéon grec en phase 1)
5. **IAristote = cœur pédagogique**, tous les flux intelligents passent par lui
6. **Dialogues RPG immersifs** pour toute prise de parole divine (jamais de Toast ni AlertDialog pour narration)
7. **Pas de plafond de progression** (infini, prestige)
8. **Chibi JRPG 3D** comme style visuel officiel

## 8. Ce qu'un agent ne doit jamais proposer

- Migrer vers Jetpack Compose
- Supprimer le mode Aventure paysage
- Remplacer le panthéon grec comme phase 1
- Simplifier les dialogues RPG en UI Android standard
- Casser IAristote comme orchestrateur IA unique
- Introduire un plafond de progression
- Changer le style visuel chibi JRPG

Toute dérive vers l'un de ces points doit être refusée et signalée.
