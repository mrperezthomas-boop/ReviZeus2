RÉVIZEUS — BLOC K
PROFILS D’ACCOMPAGNEMENT PÉDAGOGIQUE + IA ADAPTATIVE V1
Version : GOD TIER — bloc prêt à coder avec une IA
Date : 2026-03-24

════════════════════════════════════════════════════════════════════
0 — RÔLE DE CE BLOC
════════════════════════════════════════════════════════════════════

Ce bloc a pour but de doter RéviZeus d’un vrai système de profils d’accompagnement pédagogiques
sans casser l’architecture actuelle.

Le système ne doit PAS être conçu comme un diagnostic médical.
Il doit être conçu comme un réglage pédagogique noble, modifiable, rassurant et utile,
permettant à RéviZeus d’adapter :
- la forme des résumés,
- la longueur des consignes,
- le rythme des quiz,
- la densité des corrections,
- le ton des dialogues,
- certaines intensités audio/visuelles,
- et plus tard la profondeur de l’accompagnement par matière.

Ce bloc doit créer une première couche centrale, propre, exploitable, extensible,
et immédiatement branchée à l’IA et aux points d’entrée UI déjà cohérents avec RéviZeus.

════════════════════════════════════════════════════════════════════
1 — PLACE DU BLOC K DANS LE PLAN GLOBAL
════════════════════════════════════════════════════════════════════

Le BLOC K arrive après :
- A : stabilisation technique et socle transverse
- B : dialogues RPG universels
- C : audio / TTS / ducking / loadings / fonds premium
- D : Oracle premium pipeline résumé → validation → temple → quiz
- E : quiz nouvelle génération / corrections / résultats
- F : récompenses unifiées / fragments / économie / gains
- G : forge / inventaire / effets objets / craft / tri / filtres
- H : progression des temples / reconstruction / déblocages monde
- I : savoirs vivants / sous-dossiers / long press / artefacts / export
- J : extensions divines créatives (Aphrodite / Hermès / Apollon / Prométhée)

Il prépare ensuite :
- L : analytics pédagogiques avancées
- M : recommandations IA et adaptation plus profonde
- N : dashboard vivant + widgets d’insight
- O : rangs / badges / couches méta
- P : aventure structurée, quêtes, zones, boss
- Q : lore, carte du monde, intro, tension mythologique et polissage transversal

Ce bloc ne doit pas tenter de coder toute l’IA adaptative profonde d’un coup.
Il doit construire une V1 propre, robuste et déjà utile.

════════════════════════════════════════════════════════════════════
2 — OBJECTIF PRODUIT EXACT
════════════════════════════════════════════════════════════════════

Permettre à chaque héros de disposer d’un profil d’accompagnement configurable
qui influence l’IA et certaines logiques UX sans casser l’architecture actuelle.

Objectifs principaux :
- ajouter un profil stocké proprement sur le héros,
- permettre de choisir ce profil à la création du héros,
- permettre de le modifier ensuite dans les paramètres,
- centraliser sa logique dans un manager dédié,
- injecter ce profil dans GeminiManager et GodLoreManager,
- faire varier les résumés, quiz, corrections et dialogues,
- préparer une extension future vers une vraie IA adaptative profonde.

Objectifs secondaires :
- rassurer l’utilisateur sur le fait que ce choix n’est ni figé ni stigmatisant,
- rendre le système noble et immersif via une explication RPG,
- permettre à l’UX de varier sans dupliquer des conditions partout,
- préparer plus tard la combinaison avec âge, classe, humeur, matière, progression, analytics et rang méta.

════════════════════════════════════════════════════════════════════
3 — PHILOSOPHIE D’ARCHITECTURE OBLIGATOIRE
════════════════════════════════════════════════════════════════════

Règle absolue :
ne jamais disperser cette logique dans quinze Activities avec des conditions fragiles.

INTERDIT :
- if profil == tdah dans plusieurs écrans
- if profil == x dans chaque Activity
- prompts recopiés à la main dans plusieurs classes
- adaptation de l’UI faite au hasard écran par écran
- logique sensorielle et pédagogique non centralisée

OBLIGATOIRE :
- un type de profil visible côté produit,
- une config dérivée côté code,
- un manager unique qui traduit le type choisi en comportements concrets,
- quelques points d’intégration propres dans l’UI,
- une injection propre dans les managers IA.

Architecture cible recommandée :
- LearningProfileType
- LearningProfileConfig
- LearningProfileManager
- champ learningProfile dans UserProfile
- UI de sélection / modification
- helpers d’adaptation pour prompts, résumés, quiz, corrections et dialogues

4 — NOM CÔTÉ PRODUIT ET CÔTÉ CODE
════════════════════════════════════════════════════════════════════

Nom conseillé côté produit :
Profil d’accompagnement du héros

Pourquoi :
- plus élégant
- moins stigmatisant
- plus RPG
- plus cohérent avec RéviZeus
- plus rassurant pour un parent ou un enfant

Noms conseillés côté code :
- LearningProfileType
- LearningProfileConfig
- LearningProfileManager

Champs conseillés côté UserProfile :
- learningProfile: String = "STANDARD"

Extensions possibles plus tard :
- learningProfileVersion
- learningProfileLastUpdatedAt
- learningProfileSource (manual / inferred / mixed)
- learningProfileNotesJson si besoin avancé plus tard

5 — PROFILS VISIBLES V1 À IMPLÉMENTER
════════════════════════════════════════════════════════════════════

Le but n’est pas d’exposer trop de profils trop tôt.
Il faut une V1 courte, lisible, noble, utile.

Liste recommandée V1 :

5.1 STANDARD
Description produit :
RéviZeus utilise son adaptation normale selon l’âge, la classe, l’humeur,
la matière et la progression générale du héros.

Effets attendus :
- comportement par défaut
- densité standard
- ton standard
- rythme standard

5.2 CONSIGNES COURTES
Description produit :
Pour les héros qui préfèrent des consignes plus rapides, plus directes et moins chargées.

Effets attendus :
- phrases plus courtes
- moins d’informations simultanées
- consignes plus nettes
- résumés plus compacts
- corrections plus brèves

5.3 REFORMULATION GUIDÉE
Description produit :
Pour les héros qui comprennent mieux quand RéviZeus reformule, clarifie et réexplique autrement.

Effets attendus :
- vocabulaire plus clair
- plus d’analogies simples
- plus de reformulations
- plus de transitions explicites
- explications plus décomposées

5.4 RYTHME APAISÉ
Description produit :
Pour les héros qui ont besoin d’un rythme plus doux, d’un ton plus rassurant et d’une pression réduite.

Effets attendus :
- ton plus calme
- feedback moins abrupt
- corrections plus enveloppantes
- intensité de défi réduite
- allègement possible de certains effets sensoriels plus tard

5.5 GUIDAGE PAS À PAS
Description produit :
Pour les héros qui apprennent mieux quand le savoir est découpé en étapes successives.

Effets attendus :
- questions plus séquencées
- explications ordonnées
- décomposition logique plus forte
- résumés structurés en étapes
- corrections à forte composante maïeutique guidée

5.6 RENFORCEMENT POSITIF
Description produit :
Pour les héros qui gagnent en confiance quand RéviZeus valorise plus fortement les progrès.

Effets attendus :
- feedback plus encourageants
- ton plus soutenant
- valorisation fine des micro-réussites
- meilleure récupération après erreur
- intensité de punition émotionnelle réduite

5.7 DÉFI ET CHALLENGE
Description produit :
Pour les héros qui aiment être poussés davantage, avec un ton plus combatif et des défis plus francs.

Effets attendus :
- formulations plus directes
- distracteurs un peu plus denses
- encouragement au dépassement
- plus forte présence d’Arès / Zeus dans certaines formulations
- cadence plus vive si le reste du contexte le permet

Important :
Ce choix est modifiable à tout moment.
Aucun profil n’est définitif.
Aucun profil ne définit la valeur du héros.
Il sert uniquement à mieux accorder la manière d’enseigner.

════════════════════════════════════════════════════════════════════
6 — DIMENSIONS D’ADAPTATION À CENTRALISER
════════════════════════════════════════════════════════════════════

Le manager doit convertir le profil choisi en paramètres pédagogiques concrets.

Dimensions minimales à prévoir dans LearningProfileConfig :
- instructionLengthBias
- summaryDensityBias
- quizComplexityBias
- distractorDensityBias
- correctionDepthBias
- correctionWarmthBias
- maieuticStrengthBias
- guidanceStepByStepBias
- positiveFeedbackBias
- challengeBias
- sensoryIntensityBias
- pacingBias
- dialogVerbosityBias
- analogiesBias
- reformulationBias

Ces biais ne remplacent pas :
- l’âge,
- la classe,
- l’humeur,
- la matière,
- la progression réelle,
- la personnalité du dieu.

Ils s’y ajoutent comme une dimension supplémentaire.

Fichiers concernés :
- LearningProfileManager.kt

Critère de fin :
aucune logique de profil importante n’est dispersée ailleurs.

K5 — WARNING RPG LORS DE LA CRÉATION DU HÉROS
────────────────────────────────────────────────────────────────────

Objectif :
Insérer une étape ou un popup d’explication juste avant validation finale du héros.

Moment idéal :
après nom / âge / classe, juste avant validation.

Figure divine recommandée :
Prométhée en priorité
ou Athéna si la scène est plus scolaire.

Message à faire passer :
RéviZeus peut adapter la manière d’expliquer, le rythme des défis,
la longueur des consignes et certaines interactions selon la manière dont le héros apprend le mieux.
Ce choix ne définit pas sa valeur.
Il aide seulement les dieux à mieux l’accompagner.
Il pourra être modifié plus tard dans les réglages du héros.

À faire :
- créer l’étape de choix ou popup de sélection
- la rendre RPG lettre par lettre
- proposer les profils clairement
- afficher une description simple
- enregistrer le choix avant validation finale

Fichiers concernés :
- AuthActivity.kt ou flow de création héros réel
- layout / dialog associé
- DialogRPGManager / DialogRPGFragment si déjà en place
