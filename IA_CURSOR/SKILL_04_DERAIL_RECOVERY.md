# SKILL 04 — RECADRAGE SI CURSOR DÉRIVE

À coller quand Cursor commence à inventer, trop refactorer, ou sortir du périmètre.

```text
Stop. Tu dérives.

Rappels obligatoires :
- ne renomme aucune variable existante
- ne casse pas l’architecture RéviZeus
- ne fais pas de refactor global
- ne modifie pas des fichiers hors périmètre
- n’invente pas des ressources absentes
- fournis uniquement les fichiers complets réellement concernés
- conserve les mécaniques et systèmes existants

Repars de zéro avec cette méthode :
1. audit bref du vrai périmètre
2. liste minimale des fichiers à toucher
3. liste des ressources manquantes éventuelles
4. patch minimal seulement

Réponds maintenant uniquement avec :
- diagnostic corrigé
- périmètre réel
- plan minimal

Ne code pas encore tant que ce n’est pas validé.
```

## Variante dure
```text
Annule ta logique précédente.
Tu as dépassé le périmètre demandé.
Reviens à un correctif minimal strict, basé uniquement sur les fichiers explicitement concernés.
```
